    Rules & Definitions¶

    In Sumeh v2.0, the validation engine is completely decoupled from the storage layer. Rules are defined using a strictly typed dataclass.

    You extract your rule configurations using your standard data tools (Pandas, DuckDB, etc.) and map them directly into RuleDefinition objects.

    sumeh.core.rules.rule_model ¶

    Rule definition model with metadata preservation.

    Combines original RuleDef features with metadata preservation.

    RuleDefinition dataclass ¶

    RuleDefinition(
    field: Union[str, List[str]],
    check_type: str,
    value: Any = None,
    threshold: float = 1.0,
    level: str = "ROW",
    category: str = "unknown",
    execute: bool = True,
    updated_at: Optional[Any] = None,
    metadata: dict = None,
)

    Data quality rule with validation and metadata preservation.

    Features
    • Validates against RuleRegistry (manifest.json)
    • Auto-enriches category/level from manifest
    • Intelligent parsing of field/value
    • Engine compatibility validation
    • Preserves extra columns in metadata dict
